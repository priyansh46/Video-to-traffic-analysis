WHAT THIS PROJECT DOES
------------------------
This pipeline takes any highway traffic video as input and automatically
produces V2X (Vehicle-to-Everything) communication simulation results.

It answers the question:
  "If vehicles on this real highway had 802.11p radios,
   how well would they communicate?"

COMPLETE PIPELINE:
  Highway Video (.mp4)
          ↓
  Phase 1: Video inspection + camera calibration (homography)
          ↓
  Phase 2: YOLOv8 + ByteTrack vehicle detection and tracking
          ↓
  Phase 3: Auto lane detection from white lane markings
          ↓
  Phase 4: Traffic parameter extraction (flow, speed, headway, composition)
          ↓
  Phase 5: VISSIM 2026 .inpx simulation file generation
          ↓
  [MANUAL: Run simulation in PTV VISSIM 2026, export .fzp vehicle records]
          ↓
  Phase 6a: Parse .inpx geometry → link_geometry.json
          ↓
  Phase 6b: Parse .fzp + convert to absolute XY → trajectories_xy.csv
          ↓
  csv_to_sumo.py: Auto-convert trajectories to SUMO road network
          ↓
  Phase 6 OMNeT++: V2X communication simulation via Veins
          ↓
  Results: V2X-#0.sca, V2X-#0.vec (BSMs, PDR, channel metrics)

================================================================================
  VISSIM STEP (manual — requires PTV VISSIM 2026 license)
================================================================================

  1. Open outputs/vissim/highway_network.inpx in PTV VISSIM 2026
  2. Evaluation → Vehicle Records → enable output → highway_network.fzp
  3. Enable these attributes:
       SIMSEC, NO, LANE\LINK\NO, LANE\INDEX, POS, POSLAT, SPEED, VEHTYPE
  4. Simulation → Run
  5. Copy highway_network.fzp to outputs/vissim/

================================================================================
  UNDERSTANDING V2X RESULTS
================================================================================

  Each vehicle broadcasts a BSM (Basic Safety Message) every second
  containing its position, speed and heading over 802.11p radio at 5.89 GHz.

  Key metrics in V2X-#0.sca:

  Metric                  Meaning
  ──────────────────────  ─────────────────────────────────────────────────────
  generatedBSMs           Total safety messages broadcast by all vehicles
  receivedBSMs            Total safety messages received by other vehicles
  SentPackets             Total 802.11p packets transmitted
  ReceivedBroadcasts      Total successful over-the-air receptions
  TotalLostPackets        Packets that failed to transmit
  collisions:count        Radio channel collisions (0 = healthy)
  totalBusyTime           How long the radio channel was occupied
  totalDistance           Total distance driven by all vehicles
  maxSpeed                Maximum speed recorded per vehicle

  Packet Delivery Rate = receivedBSMs / generatedBSMs
    PDR > 1.0 means each message was heard by more than one neighbour
    PDR = 0   means no V2X communication happened (check sendBeacons=true)

  Zero collisions means traffic density was not high enough to saturate
  the 802.11p channel — normal for a 3-lane motorway.

================================================================================
  OUTPUT FILES
================================================================================

  V2X-#0.sca   Scalar file — one summary number per metric per vehicle.
               Read by analyze_v2x.py and export_v2x.py.
               Open in OMNeT++ IDE: File → New → Analysis file → Add data.

  V2X-#0.vec   Vector file — every single event with timestamp.
               Used for time-series graphs in OMNeT++ IDE.

  V2X-#0.vci   Index file for .vec — do not open directly.

================================================================================
  KNOWN LIMITATIONS
================================================================================

  1. Vehicles use SUMO car-following model — not exact VISSIM trajectory
     replay. Speed and flow parameters match but exact positions differ.

  2. VISSIM step requires a commercial PTV VISSIM 2026 license.

  3. Road geometry in SUMO is a simplified straight road — curves and
     junctions from the original video are not modelled.

  4. Lane spreading partially implemented — vehicles start in correct lanes
     but SUMO may consolidate them on short road segments.
