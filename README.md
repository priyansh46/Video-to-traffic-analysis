# Traffic Video to V2X Simulation Pipeline

## What this project does

This pipeline takes any highway traffic video as input and automatically produces V2X (Vehicle-to-Everything) communication simulation results. It answers the question: *"If vehicles on this real highway had 802.11p radios, how well would they communicate?"*

```
Highway Video (.mp4)
        ↓
YOLOv8 vehicle detection + tracking
        ↓
Auto lane detection
        ↓
Traffic parameter extraction
        ↓
VISSIM 2026 simulation file
        ↓
SUMO road network (auto-generated)
        ↓
OMNeT++ + Veins V2X simulation
        ↓
BSM transmission results (PDR, channel metrics)
```

## Run order — Phases 1 to 5 (Windows)

For a new video, change two lines in `configs/highway.yaml`:

```yaml
road_type: motorway_3lane    # match your road
video:
  path: "data/videos/YOUR_VIDEO.mp4"
```

Then run in order:

```bash
python scripts/phase1_preprocess.py   # ~5 min — click 4 reference points
python scripts/phase2_detect.py       # ~5-10 min — YOLOv8 detection
python scripts/phase3_lanes.py        # ~10 sec
python scripts/phase4_extract.py      # ~30 sec
python scripts/phase5_vissim.py       # ~5 sec
```

## VISSIM step (manual — requires PTV VISSIM 2026 license)

1. Open `outputs/vissim/highway_network.inpx` in PTV VISSIM 2026
2. Evaluation → Vehicle Records → enable output to `highway_network.fzp`
3. Enable attributes: `SIMSEC`, `NO`, `LANE\LINK\NO`, `LANE\INDEX`, `POS`, `POSLAT`, `SPEED`, `VEHTYPE`
4. Simulation → Run
5. Copy `highway_network.fzp` to `outputs/vissim/`

## What the V2X results mean

Each vehicle broadcasts a **BSM (Basic Safety Message)** every second containing its position, speed and heading. Nearby vehicles receive these messages over **802.11p radio** at 5.89 GHz.

| Metric | Meaning |
|---|---|
| `generatedBSMs` | Total safety messages broadcast |
| `receivedBSMs` | Total safety messages received by other vehicles |
| Packet Delivery Rate | Average neighbours that heard each BSM |
| `collisions:count` | Radio channel collisions (0 = healthy channel) |
| `totalBusyTime` | How long the radio channel was occupied |
| `ReceivedBroadcasts` | Total successful over-the-air receptions |

A PDR > 1.0 means each message was heard by more than one neighbour on average. Zero collisions means the highway traffic density was not high enough to saturate the 802.11p channel.

## Known limitations

1. Vehicles use SUMO car-following model — not exact VISSIM trajectory replay. Speed and flow match but exact positions differ slightly.
2. VISSIM step requires a commercial PTV VISSIM 2026 license.
3. Road geometry in SUMO is a simplified straight road — curves and junctions from the original video are not modelled.
4. Lane spreading is partially implemented — vehicles start in correct lanes but SUMO may consolidate them on short road segments.
